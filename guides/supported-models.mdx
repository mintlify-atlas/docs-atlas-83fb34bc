## Hardware Support Matrix

### Ascend 910B

High-performance training and inference NPU.

**Supported Engines**: vLLM, MindIE, MLGuider (select models)

**Recommended Models**:
- Qwen2.5/3 series (7B-72B)
- Quantized large models (W8A8, W4A8)
- GLM-4.5, GLM-5
- DeepSeek-R1 distill series

### Ascend 310P

Cost-effective inference NPU with dual-chip architecture.

**Supported Engines**: vLLM, MindIE, MLGuider

**Recommended Models**:
- Qwen2.5 series (7B-72B)
- DeepSeek-R1 distill series
- Qwen3-Coder-Next (MLGuider only)
- GLM-OCR (MLGuider only)

### MetaX C550

Domestic GPU alternative with CUDA-compatible interface.

**Supported Engines**: vLLM

**Recommended Models**:
- Qwen2.5/3 series
- Qwen3-235B (quantized)
- GLM-4.5/4.7
- DeepSeek series

## Quantization Formats

<Info>
  Quantized models reduce memory usage and increase inference speed with minimal accuracy loss.
</Info>

- **W8A8**: 8-bit weights and activations (balanced)
- **W4A8**: 4-bit weights, 8-bit activations (aggressive compression)

## Checking Model Availability

List all models supported by your hardware:

```bash
xw ls
```

Filter by device type:

```bash
xw device list
```

Check model details:

```bash
xw show <model-id>
```

## Model Sources

Models are sourced from:
- **ModelScope**: Official Alibaba model repository
- **HuggingFace**: Community model hub
- **Custom repositories**: Vendor-optimized variants

Model source IDs are specified in `configs/0.0.1/models.yaml` for reference.

<Note>
  Model availability is automatically filtered based on detected hardware. Only compatible models will appear in `xw ls`.
</Note>
