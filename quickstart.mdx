This guide will walk you through running your first AI model with XW CLI. You'll start the server, pull a model, and begin an interactive chat session.

<Note>
  Make sure you've completed the [installation](/installation) before proceeding.
</Note>

## Overview

Running a model with XW CLI involves three simple steps:

1. **Start the server** - Launch the XW inference server
2. **Pull a model** - Download an AI model from the registry
3. **Run the model** - Start an interactive chat session

## Step 1: Start the Server

The XW server manages model execution and provides an API for inference requests.

<Steps>
  <Step title="Start the server">
    Open a terminal and run:

    ```bash
    xw serve
    ```

    Expected output:
    ```
    [INFO] Server identity: xw-server-abc123
    [INFO] Configuration version: 1.0.0
    [INFO] Loading configurations from: /home/user/.xw/1.0.0
    [INFO] Data directory: /home/user/.xw/data
    [INFO] All configurations loaded successfully
    [INFO] Server started on http://localhost:11581
    [INFO] Press Ctrl+C to stop
    ```

    <Note>
      The server runs on port 11581 by default. Use `--port` to specify a different port.
    </Note>
  </Step>

  <Step title="Verify the server is running">
    Open a new terminal window and check the server status:

    ```bash
    xw version
    ```

    You should see both client and server versions:
    ```
    Client Version:
      Version:    1.0.0
      Build Time: 2026-03-05T10:30:00Z

    Server Version:
      Version:    1.0.0
      Build Time: 2026-03-05T10:30:00Z
    ```
  </Step>

  <Step title="Update model support (recommended)">
    After starting the server, update to the latest model configurations:

    ```bash
    xw update
    ```

    This ensures you have access to the newest models and optimizations.
  </Step>
</Steps>

<Warning>
  Keep the terminal running `xw serve` open. Opening a second terminal for the remaining commands.
</Warning>

### Server Configuration Options

Customize the server with these options:

<CodeGroup>
  ```bash Custom Port
  xw serve --port 9090
  ```

  ```bash All Network Interfaces
  xw serve --host 0.0.0.0
  ```

  ```bash Verbose Logging
  xw serve -v
  ```

  ```bash Custom Data Directory
  xw serve --data /path/to/data
  ```
</CodeGroup>

## Step 2: Pull a Model

Before running a model, you need to download it from the XW model registry.

<Steps>
  <Step title="List available models">
    See all available models:

    ```bash
    xw ls -a
    ```

    Sample output:
    ```
    NAME                    SIZE      MODIFIED         DOWNLOADED
    qwen3-8b               4.7 GB    2 weeks ago      No
    qwen2.5-7b-instruct    4.1 GB    3 weeks ago      No
    qwen2-0.5b             381 MB    1 month ago      No
    deepseek-7b            4.3 GB    2 weeks ago      No
    ```
  </Step>

  <Step title="Check detected devices">
    Verify your AI accelerator is detected:

    ```bash
    xw device list
    ```

    Example output:
    ```
    CHIP KEY  CHIP               #    PCI ADDRESS    VENDOR:DEVICE
    --------  ----               -    -----------    -------------
    npu910b   Ascend 910B        -    0000:c1:00.0   19e5:d801

    Total: 1 AI chip(s) detected (1x npu910b)
    ```
  </Step>

  <Step title="Pull a model">
    Download a model (we'll use qwen3-8b as an example):

    ```bash
    xw pull qwen3-8b
    ```

    You'll see download progress:
    ```
    Pulling qwen3-8b...
    [████████████████████████████████████] 100% | 4.7 GB/4.7 GB
    ✓ Model qwen3-8b pulled successfully
    ```

    <Note>
      Download time depends on your network speed. For your first model, we recommend starting with smaller models like `qwen2-0.5b` (381 MB).
    </Note>
  </Step>

  <Step title="Verify the model is downloaded">
    List your downloaded models:

    ```bash
    xw ls
    ```

    Output:
    ```
    NAME         SIZE      MODIFIED       DOWNLOADED
    qwen3-8b     4.7 GB    2 weeks ago    Yes
    ```
  </Step>
</Steps>

### Model Compatibility

<Warning>
  XW CLI will warn you if a model is not compatible with your AI accelerator. You can check model compatibility with:

  ```bash
  xw ls -a
  ```

  Models compatible with your hardware are indicated in the output.
</Warning>

## Step 3: Run the Model

Now you can start an interactive chat session with your model.

<Steps>
  <Step title="Start the interactive session">
    Run the model:

    ```bash
    xw run qwen3-8b
    ```

    The model will start and you'll see:
    ```
    Starting new instance: qwen3-8b
    ⠋ Waiting for instance to be ready... 5s
    ✓ Instance is ready!

    =============================================================
    Chat session started with: qwen3-8b
    Type your message and press Enter. Use '/h' or '/?' for help, '/quit' to exit.
    =============================================================

    >>>
    ```
  </Step>

  <Step title="Chat with the model">
    Start chatting! Type your message and press Enter:

    ```
    >>> Hello! Can you explain what XW CLI does?

    Assistant: XW CLI is a command-line interface for running large language
    models on domestic Chinese AI accelerators. It provides an easy-to-use
    interface similar to Ollama, but optimized for hardware like Huawei's
    Ascend NPUs. You can pull models, run them locally, and interact with
    them through a chat interface or API.

    >>>
    ```
  </Step>

  <Step title="Use chat commands">
    Access helpful commands with `/h`:

    ```
    >>> /h

      /h, /?                  Show this help
      /quit                   Exit the chat session
      /clear                  Clear conversation history
      /history                Show conversation history
      /show                   Show current configuration
      /set <param> <value>    Set a parameter:
        system-prompt <text>    Set system prompt
        temperature <0-2>       Set temperature (default: 0.7)
        top-p <0-1>             Set top-p (default: 0.9)
        max-tokens <number>     Set max tokens (default: 2048)
    ```
  </Step>

  <Step title="Exit the session">
    When you're done, exit with:

    ```
    >>> /quit
    Goodbye!
    ```
  </Step>
</Steps>

### Advanced Run Options

Customize how models run with these options:

<CodeGroup>
  ```bash Custom Alias
  xw run qwen3-8b --alias my-chat
  ```

  ```bash Specific Engine
  xw run qwen3-8b --engine vllm:docker
  ```

  ```bash Specific Devices
  xw run qwen3-8b --device 0,1
  ```

  ```bash Tensor Parallelism
  xw run qwen3-8b --tp 2
  ```
</CodeGroup>

Stop a running model:

```bash
xw stop qwen3-8b
```

### View Logs

Check logs for debugging:

```bash
xw logs qwen3-8b
```

## Troubleshooting

### Server Won't Start

**Problem**: Port already in use

```bash
[ERROR] Port 11581 is already in use
```

**Solution**: Use a different port or stop the existing server

```bash
xw serve --port 9090
```

### Model Pull Fails

**Problem**: Network timeout or connection error

**Solution**: Check your network connection and retry:

```bash
xw pull qwen3-8b
```

If the problem persists, check the model registry status:

```bash
xw ls -a
```

### Model Won't Start

**Problem**: Instance fails to start or stays in "starting" state

**Solution**: Check device availability and logs

```bash
# Check if devices are detected
xw device list

# View instance details
xw ps -a

# Check logs for errors
xw logs qwen3-8b
```

### No AI Accelerator Detected

**Problem**: `xw device list` shows no devices

**Solution**:

1. Verify driver installation:
   ```bash
   # Example for Ascend NPU
   npu-smi info
   ```

2. Check hardware is connected:
   ```bash
   lspci | grep -i accelerator
   ```

3. Restart the XW server with verbose logging:
   ```bash
   xw serve -v
   ```

### Instance Stuck in "Starting" State

**Problem**: Model takes too long to start

**Cause**: Large models can take 1-3 minutes to load, depending on hardware

**Solution**: Wait patiently. Press Ctrl+C to cancel if needed, then check logs:

```bash
xw ps -a
xw logs qwen3-8b
```

## Example Workflows

### Quick Test with Small Model

For a fast test run:

```bash
# 1. Start server
xw serve

# 2. In another terminal, pull smallest model
xw pull qwen2-0.5b

# 3. Run it
xw run qwen2-0.5b
```

### Production Setup

For production use with systemd:

```bash
# 1. Install system-wide
sudo make install

# 2. Start and enable service
sudo systemctl start xw-server
sudo systemctl enable xw-server

# 3. Pull models
xw pull qwen3-8b

# 4. Start persistent instance
xw start qwen3-8b --alias production-chat

# 5. Use API endpoint
curl http://localhost:11581/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "production-chat",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

### Multiple Models

Run multiple models simultaneously:

```bash
# Pull models
xw pull qwen2-0.5b
xw pull qwen3-8b

# Start with different aliases
xw start qwen2-0.5b --alias fast-model
xw start qwen3-8b --alias accurate-model

# Check status
xw ps

# Connect to either one
xw run fast-model
```

## Tips and Best Practices

<Note>
  **Performance Tips:**
  - Start with smaller models (< 1GB) to verify your setup
  - Use `--tp` flag for multi-device tensor parallelism on large models
  - Monitor device utilization with vendor tools (e.g., `npu-smi` for Ascend)
  - Keep configuration updated with `xw update` for latest optimizations
</Note>

<Warning>
  **Resource Management:**
  - Large models (> 7B parameters) require significant memory
  - Stop unused instances with `xw stop` to free resources
  - Check `xw ps` regularly to monitor running instances
  - Only run models compatible with your hardware
</Warning>
