The model configuration file (`models.yaml`) defines the catalog of AI models that can be deployed using XW CLI, along with their hardware compatibility and supported inference engines.

## Configuration Structure

```yaml
version: "1.0"

models:
  - model_id: qwen2.5-7b-instruct
    source_id: Qwen/Qwen2.5-7B-Instruct
    parameters: 7.6
    context_length: 131072
    supported_devices:
      ascend-910b:
        - vllm:docker
        - mindie:docker
    capabilities:
      - completion
```

## Model Definition

### Basic Fields

<AccordionGroup>
  <Accordion title="model_id (required)">
    **Type**: String  
    **Description**: Unique identifier for this model in XW CLI
    
    **Format**: Lowercase, hyphen-separated  
    **Convention**: `{family}-{size}[-{variant}]`
    
    ```yaml
    model_id: qwen2.5-7b-instruct
    ```
    
    Examples:
    - `qwen2.5-7b-instruct`
    - `qwen3-32b-w8a8`
    - `glm-4.5-w8a8`
    - `deepseek-r1-distill-qwen-7b`
    
    <Warning>Must be globally unique across all models</Warning>
  </Accordion>

  <Accordion title="source_id (required)">
    **Type**: String  
    **Description**: Model identifier on the source platform
    
    Used for model downloading and registry lookups.
    
    **Platforms**:
    - ModelScope: `{org}/{model}` (e.g., `qwen/Qwen2.5-7B-Instruct`)
    - HuggingFace: `{org}/{model}` (e.g., `Qwen/Qwen2.5-7B-Instruct`)
    - Custom registry: Full model path
    
    ```yaml
    source_id: Qwen/Qwen2.5-7B-Instruct
    ```
  </Accordion>

  <Accordion title="parameters (optional)">
    **Type**: Float  
    **Unit**: Billions of parameters
    
    **Description**: Model size in billions of parameters
    
    Used for resource estimation and display.
    
    ```yaml
    parameters: 7.6  # 7.6B parameters
    ```
    
    Examples:
    - `7.0` for 7B models
    - `32.0` for 32B models
    - `235.0` for 235B models
  </Accordion>

  <Accordion title="context_length (optional)">
    **Type**: Integer  
    **Unit**: Tokens
    
    **Description**: Maximum context window size
    
    Used for validation and resource planning.
    
    ```yaml
    context_length: 131072  # 128K tokens
    ```
    
    Common values:
    - `32768` - 32K tokens
    - `131072` - 128K tokens
    - `262144` - 256K tokens
  </Accordion>
</AccordionGroup>

### Device Compatibility

<AccordionGroup>
  <Accordion title="supported_devices (required)">
    **Type**: Map  
    **Description**: Hardware compatibility and engine priority
    
    **Structure**: `device_config_key` → `[engines]`
    
    Maps device types (from `devices.yaml`) to prioritized lists of inference engines.
    
    ```yaml
    supported_devices:
      ascend-910b:
        - vllm:docker
        - mindie:docker
      ascend-310p:
        - vllm:docker
        - mindie:docker
        - mlguider:docker
      metax-c550:
        - vllm:docker
    ```
    
    **Engine Priority**: Engines are listed in priority order. The system tries engines from first to last until one succeeds.
    
    <Info>
      The `config_key` values must match entries in `devices.yaml`
    </Info>
  </Accordion>

  <Accordion title="Engine Format">
    **Format**: `{backend}:{mode}`
    
    **Backend**: Inference engine name
    - `vllm` - vLLM inference engine
    - `mindie` - MindIE inference engine
    - `mlguider` - MLGuider inference engine
    - `omni-infer` - Omni-Infer engine
    
    **Mode**: Deployment mode
    - `docker` - Containerized deployment (recommended)
    - `native` - Native process deployment
    
    ```yaml
    - vllm:docker    # vLLM in Docker container
    - mindie:docker  # MindIE in Docker container
    - mlguider:docker # MLGuider in Docker container
    ```
    
    <Warning>Engine must have a corresponding runtime image defined in `devices.yaml`</Warning>
  </Accordion>
</AccordionGroup>

### Model Metadata

<AccordionGroup>
  <Accordion title="tag (optional)">
    **Type**: String  
    **Description**: Model variant or quantization identifier
    
    Used to distinguish between different versions of the same model.
    
    ```yaml
    tag: w8a8  # INT8 weight, INT8 activation quantization
    ```
    
    Common tags:
    - `w4a8` - W4A8 quantization (4-bit weights, 8-bit activations)
    - `w8a8` - W8A8 quantization (8-bit weights, 8-bit activations)
    - `fp16` - FP16 precision
    - `main` - Main/default variant
  </Accordion>

  <Accordion title="capabilities (optional)">
    **Type**: Array of strings  
    **Description**: Supported features and modalities
    
    Used for filtering and capability discovery.
    
    ```yaml
    capabilities:
      - completion
      - vision
      - tool_use
    ```
    
    **Standard Capabilities**:
    - `completion` - Text completion/generation
    - `vision` - Image understanding (multimodal)
    - `tool_use` - Function calling
    - `coding` - Code generation
    - `function_calling` - Structured function calls
  </Accordion>
</AccordionGroup>

## Engine Priority and Fallback

Engines are tried in order until one succeeds:

```yaml
supported_devices:
  ascend-910b:
    - vllm:docker      # Try vLLM first
    - mindie:docker    # Fall back to MindIE if vLLM fails
    - mlguider:docker  # Fall back to MLGuider if both fail
```

**Selection Logic**:

1. **Check runtime image availability**: Does the device have this engine's image?
2. **Check resource availability**: Are enough devices available?
3. **Attempt startup**: Try to start the container
4. **On failure**: Try next engine in priority list

<Info>
  Engine priority allows automatic fallback for maximum reliability.
</Info>

## Adding a New Model

<Steps>
  <Step title="Choose model_id">
    Pick a unique, descriptive identifier:
    
    ```yaml
    model_id: llama3-8b-instruct
    ```
  </Step>
  
  <Step title="Set source_id">
    Specify the model source:
    
    ```yaml
    source_id: meta-llama/Llama-3-8B-Instruct
    ```
  </Step>
  
  <Step title="Define specifications">
    Add model parameters and context length:
    
    ```yaml
    parameters: 8.0
    context_length: 8192
    ```
  </Step>
  
  <Step title="Configure device support">
    List compatible devices and engines:
    
    ```yaml
    supported_devices:
      ascend-910b:
        - vllm:docker
        - mindie:docker
    ```
  </Step>
  
  <Step title="Add capabilities">
    Specify model features:
    
    ```yaml
    capabilities:
      - completion
    ```
  </Step>
  
  <Step title="Test the model">
    Validate and deploy:
    
    ```bash
    xw server restart
    xw model list
    xw run llama3-8b-instruct
    ```
  </Step>
</Steps>

## Common Model Patterns

### Multi-Architecture Support

Define support for multiple device types:

```yaml
- model_id: qwen3-32b
  source_id: Qwen/Qwen3-32B
  parameters: 32.0
  context_length: 131072
  supported_devices:
    ascend-910b:
      - vllm:docker
      - mindie:docker
    ascend-310p:
      - vllm:docker
      - mindie:docker
      - mlguider:docker
    metax-c550:
      - vllm:docker
  capabilities:
    - completion
```

### Quantized Model Family

Provide multiple quantization options:

```yaml
# Full precision
- model_id: qwen3-32b
  source_id: Qwen/Qwen3-32B
  parameters: 32.0
  supported_devices:
    ascend-910b:
      - vllm:docker

# W8A8 quantized
- model_id: qwen3-32b-w8a8
  source_id: vllm-ascend/Qwen3-32B-W8A8
  parameters: 32.0
  tag: w8a8
  supported_devices:
    ascend-910b:
      - vllm:docker
      - mindie:docker
```

### Device-Specific Models

Models optimized for specific hardware:

```yaml
- model_id: minimax-m2.1-w4a8
  source_id: tsingmao-intelligence/MiniMax-M2.1.w4a8.ascend
  parameters: 0.0
  context_length: 131072
  supported_devices:
    ascend-910b:  # Only on Ascend 910B
      - vllm:docker
  tag: w4a8
  capabilities:
    - completion
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Model not appearing in list">
    **Possible causes**:
    - Validation error in YAML
    - Duplicate model_id
    - Missing required fields
    
    **Solution**: Check server logs for validation errors
    ```bash
    xw server restart --verbose
    ```
  </Accordion>

  <Accordion title="Model fails to deploy">
    **Possible causes**:
    - No compatible runtime image in devices.yaml
    - Device config_key mismatch
    - Insufficient device resources
    
    **Solution**: Verify device configuration
    ```bash
    xw device list
    xw model info {model_id}
    ```
  </Accordion>

  <Accordion title="Wrong engine selected">
    **Possible causes**:
    - Engine priority order incorrect
    - Runtime image missing
    
    **Solution**: Adjust engine priority in supported_devices
  </Accordion>
</AccordionGroup>
